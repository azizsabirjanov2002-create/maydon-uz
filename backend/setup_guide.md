# Инструкция по локальному запуску Maydon.uz Backend

Чтобы поднять бэкенд на Windows и развернуть созданную нами архитектуру, следуй этим шагам.

## Этап 1: Установка зависимостей (Системный уровень)

1. **Установи Node.js:**
   - Скачай LTS версию (сейчас это 20.x или 22.x) с официального сайта: [nodejs.org](https://nodejs.org/).
   - Установи её, соглашаясь со всеми параметрами по умолчанию.
   - Проверь установку: открой терминал (cmd или PowerShell) и введи:
     ```bash
     node -v
     npm -v
     ```
     *(Должны вывестись версии)*

2. **Установи PostgreSQL:**
   - Скачай установщик (версия 15 или 16) с сайта: [postgresql.org/download/windows](https://www.postgresql.org/download/windows/).
   - В процессе установки тебя попросят придумать **пароль для `postgres` суперпользователя**. Запомни его! Это будет пароль к локальной базе.
   - Оставь порт по умолчанию `5432`.
   - В конце установки **не обязательно** запускать Stack Builder, сними эту галочку.

---

## Этап 2: Настройка базы данных

1. Найди в меню пуск **pgAdmin 4** (установился вместе с PostgreSQL) и открой его.
2. Введи пароль, который ты придумал при установке.
3. Слева в дереве открой `Servers -> PostgreSQL 15/16 -> Databases`.
4. Кликни правой кнопкой мыши по `Databases` -> **Create -> Database...**
5. В поле Database name введи `maydon_db` и нажми **Save**. База создана!

---

## Этап 3: Подготовка проекта

1. **Открой папку backend в терминале**
   - Открой VS Code, нажми `File -> Open Folder` и выбери папку `C:\Users\Intel\Desktop\maydon-uz\backend`.
   - Открой встроенный терминал в VS Code (Ctrl + ~). Здесь должна быть директория: `c:\Users\Intel\Desktop\maydon-uz\backend>`.

2. **Скопируй Environment переменные**
   - В папке `backend` есть файл `.env.example`. 
   - Создай рядом файл и назови его **ровно `.env`**.
   - Скопируй содержимое из `.env.example` в `.env`. 
   - Отредактируй строку `DATABASE_URL` под свой пароль базы:
     *(Замени строку 'your-password' на тот пароль, что ты задавал при установке Postgres)*
     ```env
     DATABASE_URL="postgresql://postgres:your-password@localhost:5432/maydon_db?schema=public&timezone=Asia/Tashkent"
     ```

3. **Установи NPM пакеты**
   - Прямо в терминале введи команду:
     ```bash
     npm install
     ```
   *(Эта команда скачает папку `node_modules`. Все ошибки из-за отсутствия типов TypeScript в редакторе сразу исчезнут).*

---

## Этап 4: Развертывание архитектуры (Миграции)

Теперь нужно превратить нашу Prisma-схему в реальные таблицы БД.

1. **Сгенерируй миграцию**
   В терминале выполни:
   ```bash
   npx prisma migrate dev --name init
   ```
   *Софт поднимет таблицы `users`, `venues`, `bookings`, `audit_logs` и т.д. внутри базы.*

2. **Примени Raw SQL (GiST Exclusion)**
   Нам нужно добавить защиту слотов.
   - Открой `pgAdmin 4`. Слева выбери свою только что созданную базу `maydon_db`.
   - Нажми наверху кнопку **Query Tool** (кнопка с изображением БД и ползунком/лупой).
   - Скопируй текст из файла `backend/prisma/raw_manual_migrations/01_gist_exclude.sql`.
   - Вставь его в Query Tool и нажми Play (кнопка "Execute" или F5).
   *Защита слотов включится на стороне базы данных.*

3. **Сгенерируй тестовые данные (Seeding)**
   В терминале выполни:
   ```bash
   npm run db:seed
   ```
   *Это создаст первых тестовых пользователей, владельцев и площадки для тестов MVP.*

---

## Этап 5: Запуск сервера!

1. В терминале выполни:
   ```bash
   npm run dev
   ```
2. Если всё прошло успешно, терминал выдаст что-то вроде:
   ```
   Server listening at http://localhost:3000
   Swagger UI is available at http://localhost:3000/docs
   ```

**Готово!** Сервер работает. Ты можешь зайти на `http://localhost:3000/docs` в браузере, чтобы через графический интерфейс Swagger API (который я настроил ранее) протестировать регистрацию, логин и логику бронирования прямо сейчас.
